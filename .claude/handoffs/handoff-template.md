# Handoff Template

> Use this template when transferring work between agents. Clear handoffs prevent information loss and rework.

---

## Standard Handoff Format

```markdown
## Handoff: [From Agent] → [To Agent]

**Date**: [YYYY-MM-DD]
**Task**: [Name of the work being handed off]
**Execution Map Step**: [X of N] (reference to Orchestrator's map)

---

### Summary
[1-2 sentence description of what was completed]

### Deliverables
| Deliverable | Location | Status |
|------------|----------|--------|
| [Item 1] | [file path] | ✅ Complete |
| [Item 2] | [file path] | ✅ Complete |

### Context for Receiving Agent

**Required Reading:**
- [ ] [Document 1] — [why they need it]
- [ ] [Document 2] — [why they need it]

**Key Decisions Made:**
- [Decision 1] — [rationale]
- [Decision 2] — [rationale]

**Constraints:**
- [Constraint 1]
- [Constraint 2]

### Expected Output
[Specific description of what the receiving agent should produce]

### Acceptance Criteria
- [ ] [Criterion 1]
- [ ] [Criterion 2]
- [ ] [Criterion 3]

### Open Questions
- [ ] [Anything unresolved that the next agent should be aware of]

### Hands Off To After
[Next agent in the chain, or "QA for sign-off"]
```

---

## Handoff Checklist

Before completing ANY handoff, verify:

- [ ] All deliverables are complete and accessible
- [ ] Required reading is clearly listed
- [ ] Expected output is specific and measurable
- [ ] Acceptance criteria are verifiable
- [ ] Open questions are documented
- [ ] Receiving agent has everything needed to start without asking questions
